# Solución de problemas

## Expo Go

Debido a las dependencias nativas necesarias, no puedes usar Expo Go para ejecutar tu aplicación. Debes usar una compilación de desarrollo.

Puedes crear compilaciones de desarrollo para iOS/Android usando el siguiente comando:

```shell
npx expo prebuild
```

Puedes ejecutar la compilación de desarrollo usando el siguiente comando:

```shell
npx expo run:ios
```

```shell
npx expo run:android
```

## Error de compilación de OpenSSL en iOS

Si ves un error de compilación que menciona OpenSSL, es posible que necesites actualizar `app.json` para fijar una versión de OpenSSL que sea compatible con tu versión de Xcode.

Aquí te mostramos cómo fijar la versión de OpenSSL en `app.json`:

```json
{
  "expo": {
    "plugins": [
      "expo-build-properties",
      {
        "ios": {
          "extraPods": [
            {
              "name": "OpenSSL-Universal",
              "configurations": ["Release", "Debug"],
              "modular_headers": true,
              "version": "<OPENSSL_VERSION>"
            }
          ]
        }
      }
    ]
  }
}
```

Reemplaza `<OPENSSL_VERSION>` con la versión correcta de OpenSSL para tu versión de Xcode:

- Xcode 16: `3.3.2000`
- Xcode 15: `3.1.5004`

Asegúrate de tener `expo-build-properties` agregado en las dependencias de tu `package.json`.

## Error de minSdkVersion en Android

Si ves un error en Android que menciona `minSdkVersion`, es posible que necesites actualizar tu `app.json` para establecer la `minSdkVersion` a una versión superior.

Aquí te mostramos cómo actualizar la `minSdkVersion` en `app.json`:

```json
{
  "expo": {
    "plugins": [
      "expo-build-properties",
      {
        "android": {
            "minSdkVersion": 26
        }
      }
    ]
  }
}
```

Asegúrate de tener `expo-build-properties` agregado en las dependencias de tu `package.json`.
