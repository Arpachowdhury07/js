# Eliza

Eliza es un framework de agente de IA simple, rápido y liviano para construir agentes conversacionales flexibles, escalables y seguros.

Con el plugin de thirdweb, puedes integrar fácilmente Nebula en un agente de IA construido con Eliza para proporcionar capacidades de lectura, escritura y razonamiento cada vez más precisas en prompts relacionados con blockchain:

<OpenSourceCard
title="Nebula Plugin"
href="https://github.com/elizaOS/eliza/tree/main/packages/plugin-thirdweb"/>


## Requisitos Previos

- Crear una cuenta en thirdweb
- Node.js 23+ y pnpm 9+ instalados

## Obtener ID de Cliente y Clave Secreta

<Steps>

<Step title="Crear proyecto">

Dirígete al [panel de proyectos](https://thirdweb.com/) y crea un nuevo proyecto.

<DocImage src={NewProject} alt="Crear un nuevo proyecto"/>
  
</Step>

<Step title="Obtener claves">

Configura tu proyecto y obtén tu ID de cliente y clave secreta. Guarda tu clave secreta en un lugar seguro, ya que no es recuperable.

<Callout variant="warning" title="ID de Cliente vs Clave Secreta">
El ID de Cliente se utiliza para uso del lado del cliente y está restringido por las restricciones de dominio que configures en tu clave API. Es un identificador público que puede ser usado de forma segura en el frontend.

La clave secreta se utiliza para uso del lado del servidor o en scripts y no está restringida por las restricciones de dominio. Nunca expongas tu clave secreta en el código del lado del cliente.
</Callout>

<DocImage src={KeysSetup} alt="Obtener claves"/>

</Step>

</Steps>

## Configurar Eliza Starter

Si no has creado un proyecto con Eliza, se recomienda comenzar con el repositorio Eliza Starter:

<Steps>
<Step title="Clonar Starter">
<Callout variant='info' title='Versión'>
El plugin de Nebula está disponible solo a partir de la versión 0.1.8 de Eliza y viene incluido por defecto.
</Callout>

Clona el repositorio starter


```bash
git clone https://github.com/elizaos/eliza-starter.git
```
</Step>

<Step title="Variables de Entorno">
Crea un archivo `.env` y agrega tu clave secreta y otras variables de entorno:

```bash
THIRDWEB_SECRET_KEY=your_secret_key
```

<Callout variant="warning" title="ID de Cliente vs Clave Secreta">
El ID de Cliente se utiliza para uso del lado del cliente y está restringido por las restricciones de dominio que configures en tu clave API. Es un identificador público que puede ser usado de forma segura en el frontend.

La clave secreta se utiliza para uso del lado del servidor o en scripts y no está restringida por las restricciones de dominio. Nunca expongas tu clave secreta en el código del lado del cliente.
</Callout>
</Step>

<Step title="Instalar Dependencias">
```bash
pnpm i
```
</Step>

<Step title="Iniciar Eliza">
```bash
pnpm start
```
</Step>

</Steps>
