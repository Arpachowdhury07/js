# Lista de Verificación para Producción

Sigue las mejores prácticas al usar Engine en un entorno de producción.

### Seguridad

- La API de Engine está diseñada para ser llamada únicamente desde tu backend. Asegúrate de que tus tokens de acceso no sean accesibles desde el frontend.
  - Excepción: Los endpoints de relayer no requieren tokens de acceso. [Aprende más sobre relayers.](infrastructure/engine/guides/relayer)
- Almacena de forma segura los tokens de acceso y la clave secreta de thirdweb. Rota estas credenciales si se ven comprometidas.
- Usa etiquetas para llevar un control de tus billeteras, administradores y tokens de acceso.
- Utiliza tokens de acceso con expiraciones para otorgar acceso limitado en el tiempo.
- Revisa regularmente la lista de administradores para eliminar miembros inactivos o antiguos del equipo.

### Billeteras backend

- Recomendación: Usa una billetera respaldada por AWS KMS o Google KMS. El acceso a la billetera siempre es recuperable y las claves privadas nunca se exponen.
- Si usas una billetera local: **haz un respaldo de la clave privada**. Engine no puede recuperar claves privadas si los datos cifrados almacenados se pierden o corrompen.
- Asegúrate de que tus billeteras backend tengan fondos suficientes. Usa [webhooks de billetera](/engine/features/webhooks) para recibir alertas cuando tu saldo de gas sea bajo.
