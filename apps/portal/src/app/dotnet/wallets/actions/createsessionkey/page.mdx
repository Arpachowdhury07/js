# SmartWallet.CreateSessionKey

Concede acceso restringido al Smart Wallet a un firmante, también conocido como clave de sesión, permitiendo que dicho firmante se conecte a este Smart Wallet e interactúe con él, junto con el administrador original.

## Uso


```csharp
// Grant a session key to pk wallet (advanced use case)
var receipt = await smartWallet.CreateSessionKey(
    signerAddress: await privateKeyWallet.GetAddress(),
    approvedTargets: new List<string>() { Constants.ADDRESS_ZERO },
    nativeTokenLimitPerTransactionInWei: "0",
    permissionStartTimestamp: "0",
    permissionEndTimestamp: (Utils.GetUnixTimeStampNow() + 86400).ToString(),
    reqValidityStartTimestamp: "0",
    reqValidityEndTimestamp: Utils.GetUnixTimeStampIn10Years().ToString()
);
```

<Details summary="Parámetros">

### signerAddress

`string`: la dirección del firmante al que se le concederá acceso.

### approvedTargets

`List<string>`: los contratos con los que este firmante puede interactuar, establece a ADDRESS_ZERO para permitir todos.

### nativeTokenLimitPerTransactionInWei

`string`: cantidad de tokens nativos por transacción que este firmante puede usar, en wei.

### permissionStartTimestamp

`string`: marca de tiempo UNIX que representa la fecha en la que se concederá esta clave de sesión, establece en 0 para activarla de inmediato.

### permissionEndTimestamp

`string`: marca de tiempo UNIX que representa la fecha en la que esta clave de sesión terminará, establece en 0 para activarla de inmediato.

### reqValidityStartTimestamp

`string`: marca de tiempo UNIX que representa la fecha en la que esta solicitud firmada comenzará a ser válida y procesable.

### reqValidityEndTimestamp

`string`: marca de tiempo UNIX que representa la fecha en la que esta solicitud firmada ya no será válida ni procesable.

</Details>

<Details summary="Valor de Retorno">

### TransactionReceipt

El recibo de la transacción para esta operación de usuario, como un objeto `TransactionReceipt`.
