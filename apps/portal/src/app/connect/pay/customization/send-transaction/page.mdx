# Personalización de useSendTransaction

Aprende a personalizar la interfaz de Pay al ejecutar una transacción con `useSendTransaction`. Puedes encontrar una selección de personalizaciones populares a continuación. Para la lista completa de props, puedes [ver la referencia completa](/references/typescript/v5/SendTransactionConfig).

---

## Personalizar Tokens Compatibles

Puedes permitir que los usuarios seleccionen tu token en una cadena específica pasando un array de `SupportedTokens`. Ten en cuenta que este array reemplaza todos los tokens predeterminados listados para esa cadena.

```tsx
const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		supportedTokens: {
			"1": [
				{
					address: "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48",
					name: "USD Coin",
					symbol: "USDC",
					icon: usdcIcon,
				},
			],
		},
	},
});
```

---

## Prellenar Token de Destino

Si deseas que tus usuarios compren tu token de forma predeterminada en tu aplicación, puedes optar por prellenar el flujo de compra de Pay con `prefillBuy`.

Por ejemplo, si quisieras que los usuarios solo compren Ethereum en la red Base, podrías hacerlo de la siguiente manera:

```tsx
import { base } from "thirdweb/chains";

const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		prefillBuy: {
			token: {
				address: "0x866a087038f7C12cf33EF91aC5b1AcE6Ac1DA788",
				name: "Base ETH",
				symbol: "ETH",
				icon: "...", // optional
			},
			chain: base,
			allowEdits: {
				amount: true, // allow editing buy amount
				token: false, // disable selecting buy token
				chain: false, // disable selecting buy chain
			},
		},
	},
});
```

Si deseas prellenar una compra con un token nativo, puedes establecer la cadena sin pasar un token:

```tsx
const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		prefillBuy: {
			chain: base,
		},
	},
});
```

## Proveedor Preferido

Puedes especificar qué proveedor de onramp presentar a tus usuarios. Por defecto, elegimos un proveedor recomendado según la ubicación del usuario, el estado de KYC y la moneda. Asegúrate de que el proveedor configurado esté [disponible en tu país](../onramp-providers).

```tsx
const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		preferredProvider: "STRIPE" | "KADO" | "TRANSAK",
	},
});
```

## Deshabilitar Métodos de Pago

En algunos casos, es posible que solo quieras mostrar a los usuarios opciones de pago en fiat o criptomonedas para tus bienes o servicios en la cadena. Puedes hacer esto configurando `buyWithCrypto` o `buyWithFiat` en `false`.

#### Deshabilitar Comprar con Criptomonedas

```tsx
const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		buyWithCrypto: false,
	},
});
```

#### Deshabilitar Comprar con Fiat

```tsx
const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		buyWithFiat: false,
	},
});
```

## Apariencia

Puedes configurar la apariencia para el componente Pay, que está configurada como `"dark"` por defecto. La apariencia puede establecerse como `"dark"`, `"light"` o un objeto personalizado.

Contamos con los proveedores [`lightTheme`](/references/typescript/v5/lightTheme) y [`darkTheme`](/references/typescript/v5/darkTheme) que puedes sobrescribir para iniciar la personalización.

Puedes consultar nuestra página de [`Apariencia`](/references/typescript/v5/Theme) para obtener una vista completa de las opciones personalizables si prefieres crear una apariencia personalizada desde cero.

#### Apariencias Proporcionadas

```tsx
const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		theme: "dark", // or "light"
	},
});
```

#### Apariencia Personalizada

```tsx
import { darkTheme } from 'thirdweb/react';

// Using custom theme
const { mutate: sendTransaction } = useSendTransaction({
	payModal: {
		theme: darkTheme({ ... },
	}}
);
```
