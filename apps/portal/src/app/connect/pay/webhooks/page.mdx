# Webhooks

Pay se puede configurar para enviar eventos de webhook que notifiquen a tu aplicación cada vez que ocurra un evento en tu transacción. Pay envía una respuesta, a través de una solicitud HTTP, a cualquier URL de endpoint que hayas proporcionado en la página Equipo > Proyecto > Connect > Pay > Webhooks en el [panel de thirdweb](https://thirdweb.com/team).

## Eventos

Para escuchar eventos, crea un webhook en la página Equipo > Proyecto > Connect > Pay > Webhooks en el [panel de thirdweb](https://thirdweb.com/team). Las URLs de los webhooks deben comenzar con `https://`.

| Event               | Description                         |
| ------------------- | ----------------------------------- |
| `purchase_complete` | A transaction is confirmed onchain. |

### Compra completada

Se activa cuando una transacción se confirma en la cadena. Este evento proporciona información sobre el nuevo estado del pedido y su `transactionHash`, así como otros detalles relevantes.

## Verificación de Webhook (Recomendado)

Dado que cualquier origen externo puede realizar llamadas a tu endpoint de webhook, se recomienda verificar la firma del webhook para asegurarte de que una solicitud proviene de tu instancia de Pay.

### Verifica la Firma

El cuerpo del payload se firma con el secreto del webhook y se proporciona en el encabezado de la solicitud `X-Pay-Signature`.

Obtén el secreto del webhook para tu endpoint desde el panel de control.

Este ejemplo de código verifica si la firma es válida:


```tsx
const generateSignature = (
	body: string,
	timestamp: string,
	secret: string,
): string => {
	const payload = `${timestamp}.${body}`;
	return crypto.createHmac("sha256", secret).update(payload).digest("hex");
};

const isValidSignature = (
	body: string,
	timestamp: string,
	signature: string,
	secret: string,
): boolean => {
	const expectedSignature = generateSignature(body, timestamp, secret);
	return crypto.timingSafeEqual(
		Buffer.from(expectedSignature),
		Buffer.from(signature),
	);
};
```

### Verifica la Marca de Tiempo

La marca de tiempo del evento se proporciona en el encabezado de la solicitud `X-Pay-Timestamp`.

Este ejemplo de código verifica si el evento excede una duración de expiración determinada:


```tsx
export const isExpired = (
	timestamp: string,
	expirationInSeconds: number,
): boolean => {
	const currentTime = Math.floor(Date.now() / 1000);
	return currentTime - parseInt(timestamp) > expirationInSeconds;
};
```

### Ejemplo de endpoint para webhook

Este ejemplo de código en NodeJS escucha notificaciones de eventos en el endpoint `/webhook`:

```tsx
import express from "express";
import bodyParser from "body-parser";
import { isValidSignature, isExpired } from "./webhookHelper";

const app = express();
const port = 3000;

const WEBHOOK_SECRET = "<your_webhook_auth_secret>";

app.use(bodyParser.text());

app.post("/webhook", (req, res) => {
	const signatureFromHeader = req.header("X-Pay-Signature");
	const timestampFromHeader = req.header("X-Pay-Timestamp");

	if (!signatureFromHeader || !timestampFromHeader) {
		return res.status(401).send("Missing signature or timestamp header");
	}

	if (
		!isValidSignature(
			req.body,
			timestampFromHeader,
			signatureFromHeader,
			WEBHOOK_SECRET,
		)
	) {
		return res.status(401).send("Invalid signature");
	}

	if (isExpired(timestampFromHeader, 300)) {
		// Assuming expiration time is 5 minutes (300 seconds)
		return res.status(401).send("Request has expired");
	}

	// Process the request
	res.status(200).send("Webhook received!");
});

app.listen(port, () => {
	console.log(`Server started on http://localhost:${port}`);
});
```
