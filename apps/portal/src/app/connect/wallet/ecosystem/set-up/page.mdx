# Comenzar

Aprende a crear tu primera billetera de ecosistema, personalizar tu billetera e integrar tu primer socio de ecosistema.

<Steps>
<Step title='Obtén tu Client ID'>
En el panel de thirdweb, navega a configuración para crear una API Key y obtener un Client ID.
</Step>
<Step title='Despliega Tu Instancia'>

Visita el [panel de Ecosystem Wallets](https://thirdweb.com/team/~/~/ecosystem/create) e ingresa el nombre de tu ecosistema y elige tu logo.

<Callout variant="warning" title="Ecosystem name is Immutable">
	El nombre de tu ecosistema es inmutable y aparecerá dondequiera que tu billetera esté disponible, así que elígelo con cuidado.
</Callout>

Cuando se despliegue tu billetera de ecosistema, recibirás un **Ecosystem ID** derivado del nombre de tu ecosistema. Utilizarás este ID para especificar tu billetera durante la integración (ver Paso 4 a continuación).

<DocImage src={CreateEcosystemWallet} />
</Step>

<Step title='Configura Permisos del Ecosistema'>

Hay dos opciones de permisos dependiendo de si deseas una billetera de ecosistema abierta o exclusiva.

<DocImage src={EcosystemWalletPermissions} />

#### Opción 1: Permisos para Cualquiera

Si deseas permitir que cualquiera integre tu billetera de ecosistema, no se requieren más cambios. Puedes proteger tu ecosistema contra actores maliciosos añadiendo dominios no deseados a tu lista de bloqueo.

#### Opción 2: Permisos de Lista de Permitidos

Si deseas crear una billetera de ecosistema para socios selectos, puedes dirigirte al panel de Ecosystem Wallet, navegar a Permisos y seleccionar "Lista de Permitidos". Todas las aplicaciones y juegos agregados a tu ecosistema, incluida tu propia aplicación, recibirán un Partner ID único.

**Agregar Aplicaciones a tu Lista de Permitidos**

Para cada aplicación en tu lista de permitidos, deberás especificar un nombre, una lista de dominios permitidos, configuraciones de firma de billetera y un ID de paquete opcional (para aplicaciones móviles). La lista de dominios determinará dónde un socio podrá mostrar tu ecosistema como una opción de inicio de sesión. Una vez que se agregue un socio, se generará un **Partner ID**.

<Callout title="Agregar tu propia aplicación como socio" variant="info">
	Deberás agregar tus propios dominios e IDs de paquetes a tu lista de permitidos para poder iniciar sesión a través de tu billetera de ecosistema en tu aplicación.
</Callout>

<DocImage src={AddAPartner} />
</Step>

<Step title='Integrar tu Billetera de Ecosistema'>

Una vez que hayas configurado tu billetera de ecosistema, puedes especificarla utilizando tu **Ecosystem ID**.

```ts
import { ecosystemWallet } from "thirdweb/wallets";

// Crear una billetera para un ecosistema "Cualquiera" sin restricciones
const wallet = ecosystemWallet("ecosystem.your-ecosystem-id");
```

Si configuraste tu billetera de ecosistema para Lista de Permitidos, también deberás especificar tu **Partner ID**.

```ts
import { ecosystemWallet } from "thirdweb/wallets";

// Crear una billetera para ecosistemas "Lista de Permitidos" restringidos a socios
const wallet = ecosystemWallet("ecosystem.your-ecosystem-id", {
	partnerId: "...",
});
```

Una vez que hayas instanciado tu billetera de ecosistema, puedes agregarla a tu flujo de inicio de sesión.

```tsx
<ConnectButton
	client={THIRDWEB_CLIENT}
	wallets={[wallet]} // Pasa una billetera no restringida o restringida al ConnectButton
	{...props}
/>
```

Tú y tus socios pueden integrar tu billetera de ecosistema como un sistema de inicio de sesión completo o como una sola opción de inicio de sesión. Puedes leer más en nuestra [guía de integración de socios](/connect/ecosystems/integrating-partners).

</Step>

</Steps>
