# Billeteras Externas

thirdweb es compatible de forma nativa con más de 500 billeteras y con todas las billeteras que implementan EIP-6963 (Descubrimiento de Proveedores Multi-inyectados). Para la lista completa, consulta [esta lista](/typescript/v5/supported-wallets).

Puedes usar una billetera externa de muchas maneras. Hemos delineado estos diferentes casos de uso a continuación:


| USO | IDEAL PARA  |
|----------|---------- |
| Conectar al sitio | Aplicaciones que solo necesitan conectividad de billetera web3 |
| Autenticarse en tu sitio | Aplicaciones que requieren autenticación con usuarios web3 existentes |
| Autenticarse en una billetera thirdweb (ya sea en la aplicación o del ecosistema) | Aplicaciones que buscan proporcionar una experiencia web3 fluida con mínimas interrupciones y solicitudes |

¡La buena noticia es que thirdweb cubre cada uno de estos casos de uso!

Sigue leyendo para descubrir cómo se hace:

## Conectando Billeteras Externas

Las billeteras externas son perfectas para aplicaciones que quieren integrarse con usuarios web3 existentes. Estos usuarios generalmente ya tienen billeteras como MetaMask, Coinbase Wallet o Rainbow instaladas y están familiarizados con los conceptos de web3.

<Callout variant="warning" title='Limitaciones'>
  Tu backend no sabrá nada sobre la billetera del usuario. 
  Esto está bien si la billetera del usuario se usa para firmar transacciones e interactuar directamente con la blockchain.

  Si necesitas que tu backend conozca la billetera del usuario, consulta [SIWE](#authenticating-with-external-wallets).
</Callout>

## Autenticación con Billeteras Externas

La autenticación con billeteras externas usando Sign-In with Ethereum (SIWE) permite que tu backend verifique de manera segura la propiedad de la billetera del usuario. Esto es esencial para aplicaciones que requieren datos específicos del usuario o acciones autenticadas.

<Callout variant="info" title='Diferencia clave'>
  A diferencia de las conexiones simples de billetera, la autenticación SIWE crea una sesión entre tu backend y la billetera del usuario, permitiendo llamadas API seguras y autenticadas.

  Sin embargo, los usuarios aún necesitarán firmar solicitudes cuando interactúen con la blockchain a través de sus billeteras. Si quieres una experiencia web3 invisible, considera usar [billeteras thirdweb](#authenticating-with-external-wallets-into-thirdweb-wallets).
</Callout>

## Autenticación con Billeteras Externas en Billeteras thirdweb

Las billeteras thirdweb proporcionan una experiencia web3 fluida con mínimas solicitudes e interrupciones. Esto te dará, como desarrollador, control total sobre cómo el usuario interactúa con tu aplicación.

Sin embargo, ten en cuenta que la billetera que interactúa con tu aplicación será una billetera thirdweb, no la billetera externa del usuario. Si bien proporcionamos métodos para exportar claves y ver cómo se vinculan la billetera externa y la billetera thirdweb, si tu aplicación requiere la billetera externa del usuario, deberías usar [SIWE](#authenticating-with-external-wallets) en su lugar.
