# Descripción General

Las billeteras In-App ya permiten los métodos de inicio de sesión más populares, pero también brindan a los desarrolladores de aplicaciones la flexibilidad de usar billeteras In-App con cualquier método de autenticación. Si tienes un usuario autenticado válido, deberías poder crear una billetera In-App para él, sin importar el método de autenticación utilizado.

<DocImage src={EWSpeedRacer} />

Esto significa que los desarrolladores de aplicaciones ahora pueden:

- **Crear billeteras In-App para usuarios utilizando un servicio de autenticación existente**. Por ejemplo, si tienes un juego donde los jugadores inician sesión con su nombre de usuario y contraseña, ahora puedes crear billeteras fácilmente cuando se registren.
- **Integrarse con cualquier proveedor de inicio de sesión social**. Por ejemplo, si tienes un juego donde quieres permitir que los usuarios inicien sesión con sus credenciales de Steam o Epic Games, ahora puedes usar billeteras In-App para habilitar estas experiencias.
- **Utilizar billeteras In-App en entornos que no sean de frontend**. Por ejemplo, podrías autenticar usuarios con SSH y usar billeteras In-App con herramientas de línea de comandos (CLI).
- **Construir experiencias de autenticación completamente personalizadas**. Por ejemplo, podrías pedir a los usuarios que verifiquen sus credenciales con 2FA o claves de acceso antes de considerarlos autenticados y aprovisionar billeteras para ellos.
