# Managed Account Factory

El contrato **Managed Account Factory** se usa para desplegar billeteras inteligentes [ERC-4337](https://eips.ethereum.org/EIPS/eip-4337s) actualizables con control de permisos basado en roles.

La billetera inteligente `ManagedAccount` es un contrato inteligente **actualizable** escrito en el [patrón de contrato dinámico](https://github.com/thirdweb-dev/dynamic-contracts).

Los desarrolladores de aplicaciones pueden emitir billeteras inteligentes `ManagedAccount` programáticamente desplegando un contrato inteligente `ManagedAccountFactory`.

El administrador del contrato `ManagedAccountFactory` puede enviar actualizaciones a todas las `ManagedAccounts` que crea. Este administrador no requiere permisos sobre las billeteras inteligentes `ManagedAccount` para hacerlo.
Una actualización de las características de `ManagedAccount` se aplica a todas las billeteras inteligentes `ManagedAccount` creadas por la fábrica.
Esta es la billetera adecuada para los desarrolladores que anticipan realizar actualizaciones en las billeteras de sus usuarios.

Las billeteras inteligentes de thirdweb tienen las siguientes características por defecto:

- Tener múltiples firmantes con permisos diferentes
- Ejecutar transacciones (simples y por lotes).
- Enviar y recibir tokens nativos.
- Enviar y recibir NFTs ERC-721 y ERC-1155.
- Multillamadas.
- Almacenar metadatos del contrato.
