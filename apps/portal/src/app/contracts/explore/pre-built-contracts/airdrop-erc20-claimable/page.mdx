# Airdrop ERC-20 (Reclamable)

El contrato `AirdropERC20Claimable` es adecuado para usar cuando deseas hacer un airdrop de tokens ERC20 a una lista de direcciones de destinatarios, donde se espera que estos destinatarios reclamen individualmente su propio airdrop.

Al desplegar este contrato, debes especificar:

- La dirección del token ERC20 que se está haciendo airdrop.
- La cantidad total de tokens a distribuir.
- La dirección del propietario de los tokens desde donde se transferirán los tokens en el airdrop.
- La marca de tiempo UNIX después de la cual el período de airdrop expirará.
- Una cantidad máxima reclamable por billetera (para reclamar el airdrop).
- Un merkle root de lista permitida opcional para los destinatarios que tienen permiso para reclamar el airdrop.

Al reclamar un airdrop, un destinatario especifica:

- La dirección a la que se transferirá el airdrop reclamado.
- La cantidad total a reclamar.
- La prueba de inclusión en el merkle root de la lista permitida, si existe.
