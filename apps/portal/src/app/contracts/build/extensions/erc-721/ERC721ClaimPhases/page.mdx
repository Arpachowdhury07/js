# ERC721ClaimPhases 

```solidity
import "@thirdweb-dev/contracts/eip/ERC721.sol";
import "@thirdweb-dev/contracts/extension/Drop.sol";
```

Permite que otras billeteras reclamen/minteen tokens del contrato bajo los criterios de condiciones de reclamo implementando las extensiones [`ERC721`](/contracts/build/extensions/erc-721/ERC721) y [`Drop`](/contracts/build/extensions/general/Drop).

A diferencia de [`ERC721ClaimCondition`](/contracts/build/extensions/erc-721/ERC721ClaimConditions), esta extensión te permite establecer _múltiples_ [fases de reclamo](/glossary/claim-phases).

Esta es una extensión que se puede detectar en el panel si el contrato inteligente implementa las extensiones [`ERC721`](/contracts/build/extensions/erc-721/ERC721) y [`Drop`](/contracts/build/extensions/general/Drop).

## Uso

Este es un ejemplo de contrato inteligente que demuestra cómo heredar de las extensiones que componen la extensión detectable `ERC721ClaimPhases`.
También se detallan las funciones que están disponibles para ser (opcionalmente) sobrescritas.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "@thirdweb-dev/contracts/eip/ERC721A.sol";
import "@thirdweb-dev/contracts/extension/LazyMint.sol";
import "@thirdweb-dev/contracts/extension/Drop.sol";

contract Contract is ERC721A, LazyMint, Drop {
    constructor(
        string memory _name,
        string memory _symbol
    )
        ERC721A(
            _name,
            _symbol
        )
    {}

    function _canLazyMint() internal view override returns (bool) {
        // Tu implementación personalizada aquí
    }

    /// @dev Runs before every `claim` function call.
    function _beforeClaim(
        uint256 _tokenId,
        address _receiver,
        uint256 _quantity,
        address _currency,
        uint256 _pricePerToken,
        AllowlistProof calldata _allowlistProof,
        bytes memory _data
    ) internal virtual override {
      // Your custom implementation logic here
    }

    /// @dev Runs after every `claim` function call.
    function _afterClaim(
        uint256 _tokenId,
        address _receiver,
        uint256 _quantity,
        address _currency,
        uint256 _pricePerToken,
        AllowlistProof calldata _allowlistProof,
        bytes memory _data
    ) internal virtual override {
      // Your custom implementation logic here
    }

    /// @dev Collects and distributes the primary sale value of tokens being claimed.
    function _collectPriceOnClaim(
        address _primarySaleRecipient,
        uint256 _quantityToClaim,
        address _currency,
        uint256 _pricePerToken
    ) internal virtual override {
        if (_pricePerToken == 0) {
            return;
        }

        uint256 totalPrice = (_quantityToClaim * _pricePerToken) / 1 ether;
        require(totalPrice > 0, "quantity too low");

        if (_currency == CurrencyTransferLib.NATIVE_TOKEN) {
            require(msg.value == totalPrice, "Must send total price.");
        }

        address saleRecipient = _primarySaleRecipient;
        CurrencyTransferLib.transferCurrency(
            _currency,
            msg.sender,
            saleRecipient,
            totalPrice
        );
    }

    /// @dev Transfers the tokens being claimed.
    function _transferTokensOnClaim(address _to, uint256 _quantityBeingClaimed)
        internal
        virtual
        override
        returns (uint256)
    {
        _mint(_to, _quantityBeingClaimed);
        return 0;
    }

    /// @dev Checks whether platform fee info can be set in the given execution context.
    function _canSetClaimConditions()
        internal
        view
        virtual
        override
        returns (bool)
    {
        return msg.sender == owner();
    }
}
```
