# ERC20Vote

El contrato inteligente `ERC20Vote` implementa el estándar [ERC20](https://eips.ethereum.org/EIPS/eip-20).
También es compatible con [EIP-2612](https://eips.ethereum.org/EIPS/eip-2612), lo que permite que las aprobaciones se realicen mediante firmas, y con el
contrato [ERC20Votes](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/master/contracts/token/ERC20/extensions/ERC20Votes.sol);
que proporciona funcionalidad de votación y delegación, junto con delegación por firma.

```solidity
import "@thirdweb-dev/contracts/base/ERC20Vote.sol";
```

<GithubButtonLink href="https://github.com/thirdweb-dev/contracts/blob/main/contracts/base/ERC20Vote.sol" />

## Uso

Importa el contrato e inhíerelo.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "@thirdweb-dev/contracts/base/ERC20Vote.sol";

contract MyToken is ERC20Vote {
	  constructor(
        address _defaultAdmin,
        string memory _name,
        string memory _symbol
    )
        ERC20Vote(
            _defaultAdmin,
            _name,
            _symbol
        )
    {}
}
```

## Funciones para sobrescribir

Las siguientes funciones han sido implementadas en este contrato y están disponibles para ser sobrescritas con lógica personalizada:

<Details id="mintTo" summary="mintTo">

```solidity
/**
 *  @notice          Permite que una dirección autorizada acuñe tokens a un destinatario.
 *  @dev             La lógica en la función `_canMint` determina si la persona que llama está autorizada para acuñar tokens.
 *
 *  @param _to       El destinatario de los tokens a acuñar.
 *  @param _amount   Cantidad de tokens a acuñar.
 */
function mintTo(address _to, uint256 _amount) public virtual {
    require(_canMint(), "Not authorized to mint.");
    require(_amount != 0, "Minting zero tokens.");

    _mint(_to, _amount);
}

```

</Details>

<Details id="burn" summary="burn">

```solidity
/**
 *  @notice          Permite que un propietario queme una cantidad determinada de sus tokens.
 *  @dev             La persona que llama debe poseer la cantidad `_amount` de tokens.
 *
 *  @param _amount   La cantidad de tokens a quemar.
 */
function burn(uint256 _amount) external virtual {
    require(balanceOf(_msgSender()) >= _amount, "not enough balance");
    _burn(msg.sender, _amount);
}
```
