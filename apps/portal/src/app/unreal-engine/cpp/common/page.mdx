# Común

`ThirdwebCommon` define varios enums y structs comunes que se utilizan en todo el SDK de Unreal Engine de Thirdweb. Representan diversos estados y resultados para operaciones relacionadas con billeteras, contratos inteligentes y proveedores de autenticación.

## Enums

### **EFunctionResult**
Representa el resultado de una llamada de función genérica.
- `Success`: La operación se completó correctamente.
- `Failed`: La operación falló.

### **ESmartWalletDeployedFunctionResult**
Representa el estado de despliegue de una billetera inteligente.
- `Deployed`: La billetera inteligente está desplegada.
- `NotDeployed`: La billetera inteligente no está desplegada.
- `Failed`: La verificación del estado de despliegue falló.

### **EOTPVerificationFunctionResult**
Representa el resultado de una verificación de OTP (One-Time Password).
- `Verified`: El OTP fue verificado correctamente.
- `Retry`: La verificación del OTP falló, pero puede ser reintentada.
- `Failed`: La verificación del OTP falló sin opción de reintento.

### **EThirdwebOAuthProvider**
- Representa los proveedores OAuth compatibles.
- `Google`: Representa a Google como proveedor OAuth.
- `Apple`: Representa a Apple como proveedor OAuth.
- `Facebook`: Representa a Facebook como proveedor OAuth.
- `Discord`: Representa a Discord como proveedor OAuth.
- `Farcaster`: Representa a Farcaster como proveedor OAuth.
- `Telegram`: Representa a Telegram como proveedor OAuth.
- `Line`: Representa a Line como proveedor OAuth.
- `X`: Representa a X como proveedor OAuth.
- `Coinbase`: Representa a Coinbase como proveedor OAuth.

### **EThirdwebOTPMethod**
- Representa los métodos de autenticación OTP compatibles.
- `Email`: Autenticación por correo electrónico.
- `Phone`: Autenticación por teléfono.

## Structs

### FSigner

`FSigner` representa un firmante en el contexto de una billetera inteligente dentro del SDK de Thirdweb. El firmante es responsable de firmar transacciones y gestionar la aprobación de objetivos y límites en tokens nativos.

#### Propiedades
- `Address`: La dirección de Ethereum del firmante.
- `ApprovedTargets`: Un array de direcciones de contrato aprobadas con las que el firmante puede interactuar.
- `NativeTokenLimitPerTransaction`: El límite de tokens nativos (por ejemplo, ETH) que se pueden gastar por transacción.
- `StartTime`: El tiempo de inicio de los permisos del firmante, especificado como un `FDateTime`.
- `EndTime`: El tiempo de finalización de los permisos del firmante, especificado como un `FDateTime`.

#### Funciones de Clase
- `FromJson(JsonObject)`: Crea una instancia de `FSigner` a partir de un objeto JSON.
